Overview

Genesis is a versatile framework designed to standardize and simplify process creation and interprocess communication targeted for embedded domain. With Genesis, developers can achieve unparalleled efficiency, flexibility, and scalability for complex systems.

It contains c++ header only libray which makes it easier to integrate into any project.

Key Features

Standardized Process Architecture

  • Each class inheriting from the Genesis framework automatically gains a dedicated mailbox and a separate thread.

  • Supports receiving and handling data in any generic format.

  • Simplifies synchronization, reducing development complexity.

Built-In Publisher & Subscriber Threads

  • A publisher thread handles serialization and sends messages to a specific port.

  • A subscriber thread connects to targeted ports, deserializes messages, and routes them to the intended recipient thread.

  • Enables seamless communication across processes.

High Scalability

  • Supports sophisticated concurrent operations with minimal developer effort.

  • Processes can scale effortlessly by leveraging Genesisâ€™ thread-per-class model and efficient message handling.

Flexible Data Handling

  • Use JSON for seamless serialization and deserialization of dynamic and structured data.

  • Supports passing complex, structured, and dynamic data between threads and processes

Interprocess Communication Powered by ZMQ

  • Uses ZMQ in publisher-subscriber mode for robust and high-performance message passing.

  • Ensures reliable communication between processes within or across systems.

How It Works

Thread-Per-Class Architecture

Publisher-Subscriber Design

Each publisher thread binds to a unique port and publishes message to it.

Each subscriber thread subscribes to ports it is interested in .

Any process can communicate to any other process.
